# 🔀 vDS(Distributed Switch) 설계

## 1. 개요 및 프로젝트 내 역할

주센터는 단일 물리 서버 위에 VMware ESXi 7.0U3를 올리고, 그 안에서 **중첩 가상화(Nested Virtualization)** 로 ESXi 클러스터 3개(Core / DB / CI·CD)를 운영하는 구조입니다. 이 세 클러스터의 네트워크를 **하나의 vSphere Distributed Switch(vDS)** 로 통합하여, 클러스터 간 포트그룹 정책을 일관되게 관리하고 vMotion·NIOC를 단일하게 제어합니다.

표준 vSwitch(VSS)가 아닌 vDS를 선택한 이유는, 여러 호스트(중첩 ESXi)에 걸친 포트그룹·VLAN 정책을 vCenter에서 중앙 집중 관리할 수 있고, 호스트별로 설정을 따로 맞추는 휴먼 에러를 제거할 수 있기 때문입니다.

<br />

## 2. 설계 방향 — 단일 DVS 통합

3개 클러스터를 별도 DVS로 분리하지 않고 **단일 DVS로 통합**했습니다. 그 판단 근거는 다음과 같습니다.

- **단일 사이트·동일 VLAN 체계:** 주센터는 물리적으로 한 사이트이고 VLAN 설계도 통합되어 있어, DVS를 나눌 경우 동일한 VLAN/포트그룹을 중복 정의해야 합니다.
- **중첩 가상화의 물리 NIC 제약:** 중첩 환경 특성상 클러스터별로 물리 NIC을 독립 할당하기 어렵습니다. 물리 NIC을 공유하는 이상 DVS 분리의 실익이 적습니다.
- **클러스터 간 vMotion·NIOC 통합 이득:** 단일 DVS여야 클러스터를 넘나드는 vMotion이 자연스럽고, Network I/O Control도 한 곳에서 통합 적용됩니다.

결론적으로 이 환경에서는 **포트그룹 일관성·vMotion·NIOC 통합 이득이 fault domain 분리 이득보다 크다**고 판단했습니다.

<br />

## 3. 단일 DVS의 트레이드오프

분리하지 않은 대신 감수한 단점도 명확히 인지하고 있습니다.

- **장애 도메인 미분리:** DVS 업그레이드 시 3개 클러스터 전체가 영향권에 들어갑니다. 따라서 업그레이드 시에는 각 클러스터의 DRS를 수동 모드로 전환한 뒤 순차 진행해야 합니다.

**분리가 정당화되는 예외 조건**(이번 프로젝트에는 해당 없음):

- DMZ / PCI-DSS 등 규제상 네트워크 격리가 강제되는 경우
- 클러스터별로 물리 NIC 스펙이 상이하여 업링크 정책을 다르게 가져가야 하는 경우

## 4. 주요 설정

### 4-1. 관리 vmk(vmk0) 이전 시 주의

VSS에서 vDS로 마이그레이션할 때 가장 위험한 지점은 **관리용 vmkernel(vmk0)** 의 이전입니다.

- 관리 vmk(vmk0)는 반드시 **관리망 전용 포트그룹으로만** 이전합니다.
- VM 트래픽용 VLAN 포트그룹과 **혼용 금지**입니다.

> ⚠️ **TRAP**
> 관리망 vmk에 VM용 VLAN 태그를 잘못 적용하면 vCenter ↔ 호스트 통신이 단절됩니다. 이 경우 vDS의 자동 롤백(약 30초)이 발동하여 이전 상태로 복구되지만, 애초에 관리망 PG와 VM PG를 명확히 분리해 두는 것이 안전합니다.

### 4-2. 중첩 ESXi용 포트그룹의 보안 정책

중첩 ESXi VM에 NIC을 추가하거나 그 위에서 또 가상 스위치를 돌릴 때는, **상위(물리 ESXi) 포트그룹의 보안 정책**을 열어줘야 내부 VM들의 트래픽이 정상 전달됩니다.

해당 포트그룹에서 다음 3가지를 **Accept** 로 설정합니다.

- Promiscuous Mode
- Forged Transmits
- MAC Address Changes

이를 누락하면 중첩된 VM의 MAC 주소로 향하는 트래픽이 상위 가상 스위치에서 차단됩니다.

<br />

## 5. 동작 흐름 및 트러블슈팅

### 정상 동작 흐름

1. vCenter에서 단일 DVS를 생성하고 VLAN별 분산 포트그룹을 정의합니다.
2. 3개 클러스터의 중첩 ESXi 호스트를 동일 DVS에 추가합니다.
3. 관리 vmk(vmk0)는 관리망 전용 PG로, VM 트래픽은 각 VLAN PG로 분리 배치합니다.
4. 중첩 환경을 위해 상위 포트그룹의 보안 정책(Promiscuous / Forged Transmits / MAC Changes)을 Accept로 설정합니다.

### 트러블슈팅

| 증상 | 원인 | 조치 |
| --- | --- | --- |
| 마이그레이션 직후 vCenter-호스트 통신 단절 | 관리 vmk에 VM용 VLAN 태그 오적용 | vDS 자동 롤백(30초) 대기 후, 관리망 전용 PG로 재이전 |
| 중첩 VM의 네트워크 통신 불가 | 상위 포트그룹 보안 정책 미개방 | Promiscuous / Forged Transmits / MAC Address Changes를 Accept로 변경 |
| DVS 업그레이드 시 클러스터 광범위 영향 | 단일 DVS의 장애 도메인 미분리 | 업그레이드 전 각 클러스터 DRS를 수동 모드로 전환 후 순차 진행 |
